# Code of Conduct

## Our commitment

This project is a small, focused developer tool. Everyone who participates — by opening issues, submitting pull requests, or joining discussions — is expected to do so respectfully and in good faith.

## Expected behaviour

- Be direct and constructive. Disagreeing with an approach is fine; dismissing the person behind it is not.
- Assume good intent. Most mistakes are honest ones.
- Give useful feedback. "This is wrong" is not feedback. "This breaks X because Y, here's an alternative" is.
- Respect that maintainers have limited time. A slow response is not a rejection.

## Unacceptable behaviour

- Harassment, personal attacks, or discriminatory language of any kind.
- Deliberately hostile or demeaning comments.
- Sustained disruption of discussions.
- Submitting intentionally malicious code or issues.

## Scope

This code of conduct applies to all project spaces: GitHub issues, pull requests, discussions, and any other forum used for project communication.

## Enforcement

Violations can be reported to the project maintainers via GitHub. Reports will be reviewed promptly and handled with discretion. Maintainers reserve the right to remove comments, close issues, or ban contributors who repeatedly act in bad faith.

## Ethics

This tool is built to help developers work more efficiently with the .NET test ecosystem. It is not intended to be used to circumvent, suppress, or obscure test results, or to misrepresent software quality. Contributions that serve those ends will be declined.
